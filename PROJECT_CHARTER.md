# Superloop Workshop Project Charter

**Version:** v0.1  
**Date:** 2026-07-10  
**Status:** Foundational

## Purpose

Superloop Workshop provides an independent environment for testing whether recurrent modular units, pneumatic or fluid-like computation, local barriers, and distributed equilibrium can form a useful computational architecture.

The project is permitted to be speculative. It is not permitted to disguise speculation as validation.

## Foundational questions

1. Can one self-contained recurrent unit compose indefinitely through a stable, standardized interface?
2. Can the same interlock act as both a barrier and a conduit without requiring a central scheduler?
3. Can bounded flow, backpressure, and reciprocal feedback maintain useful equilibrium under changing workloads?
4. Can field topology perform operating functions such as scheduling, isolation, recovery, and accounting?
5. Can neural-style integration and learning occur locally without weakening hard safety invariants?
6. Which pneumatic mechanisms are computationally substantive, and which remain analogies?
7. Under what workloads is a centralized system simpler, safer, or more efficient?

## Independence covenant

Superloop Workshop is not an Aurora module, branch, successor, or canonical subsystem.

Aurora-derived experience may enter only as one of the following:

- a cited design lesson;
- a deliberately reimplemented general pattern;
- a comparative baseline;
- a separately documented bridge proposal.

It may not enter as silent authority, inherited canon, undeclared dependency, transferred identity, or presumed compatibility.

## Promotion ladder

Concepts progress through the following states:

1. **Seed** — a named intuition or research question.
2. **Formalized** — defined variables, interfaces, invariants, and non-claims.
3. **Prototyped** — executable behavior exists with reproducible inputs.
4. **Tested** — success and failure cases have been run against declared metrics.
5. **Candidate** — evidence supports architectural use within Superloop.
6. **Adopted** — accepted as part of the Superloop reference architecture.
7. **Retired** — preserved with rationale and replacement lineage.

No stage is implied by eloquence, visual appeal, or conceptual resonance.

## Initial scope

- Formal CBL and interlock semantics
- Bounded queues and credit-based admission
- Pressure, resistance, flow, and dissipation models
- Local feedback and topological barriers
- Deterministic small-field simulation
- Fault injection and recovery measurement
- CBF–pneumatic bridge analysis

## Deferred scope

- Production deployment
- Autonomous self-modification
- Unbounded or uncontrolled network growth
- Claims of physical thermodynamic equivalence
- Claims of consciousness or biological fidelity
- Integration into Aurora or any external canonical runtime

## Success criterion for the first cycle

The first cycle succeeds when a reproducible simulation demonstrates either:

- a bounded, live CBF configuration with clearly measured tradeoffs against simpler baselines; or
- a well-supported failure showing which assumptions of the architecture must be revised.

Both outcomes advance the research.
