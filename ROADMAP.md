# Superloop Workshop Research Roadmap

## Phase 0 — Foundation

- Establish independent project identity and governance.
- Preserve the CBF and Biological Pneumatic Engine white papers with provenance notes.
- Register foundational concepts and their current evidence status.
- Define the minimal interlock contract.

**Exit condition:** repository foundation reviewed and merged.

## Phase 1 — Bridge formulation

- Map pressure, flow, resistance, valves, oscillators, memory, and cost across the two source architectures.
- Separate exact correspondences from analogies and speculative extensions.
- Resolve whether the Biological Pneumatic Engine is a local constitutive model, an experimental substrate, or only a research precedent.
- Produce a falsifiable bridge architecture.

**Exit condition:** bridge document v0.1 with explicit non-equivalences and testable hypotheses.

## Phase 2 — Minimal deterministic field

- Implement a three-ring discrete-event simulation.
- Add bounded conduits, reciprocal feedback, credits, leases, and provenance.
- Compare centralized scheduling, global barriers, and local CBF interlocks.
- Test normal operation, saturation, slow neighbor, failed neighbor, and circular wait.

**Exit condition:** reproducible benchmark report with preserved raw traces.

## Phase 3 — Pneumatic constitutive layer

- Introduce calibrated pressure, resistance, storage, and dissipation variables.
- Distinguish productive flow from costly dissipation.
- Compare discrete-token and continuous-fluid approximations.
- Test whether the pneumatic layer adds explanatory or operational value.

**Exit condition:** quantitative evidence that supports adoption, revision, or rejection.

## Phase 4 — Adaptive and neural behavior

- Add local permeability updates and phase coupling.
- Keep hard invariants outside the learned parameter space.
- Test feedback capture, oscillation, starvation, and false equilibrium.
- Compare fixed, adaptive, and equilibrium-learning configurations.

**Exit condition:** adaptive behavior improves declared metrics without weakening safety.

## Phase 5 — Distributed deployment

- Run across separate hosts or devices.
- Introduce real latency, partition, churn, clock drift, and heterogeneous capacity.
- Measure coordination overhead and fault radius.
- Evaluate whether any central services remain justified.

**Exit condition:** evidence-backed candidate reference architecture or documented termination rationale.
